5. Hours 4-8 — Decay Engine (the Time Machine math)
Goal: a fast NumPy-vectorized retention computation that powers the dashboard and the Time Machine

The math (from your blueprint)
Retention R(t) = exp(-hours_since_access / (S × k × 24)) where:

(cid:127) S = stability_S (chunk attribute, increases with reviews)
(cid:127) k = complexity_k (chunk attribute, complexity modifier)
(cid:127) Multiplying by 24 converts S (in days) to hours

Prompt 12 — RRF fusion
PROMPT TO PASTE INTO YOUR LLM
In intelligence/search.py, add the RRF fusion function.
def reciprocal_rank_fusion(
*ranked_lists: list[tuple[str, float]], # variable number of (chunk_id, score)
lists
k: int = 60
) -> list[tuple[str, float]]
Combines multiple ranked lists into one using RRF. Returns sorted by RRF score
desc.
Algorithm (one-line formula from Cormack et al. 2009):
For each chunk_id appearing in any ranked list:
rrf_score = sum(1 / (k + rank + 1) for each list it appears in, using its rank)
where rank is 0-indexed position in that list.
The k=60 default is the SIGIR 2009 paper's value and what Elasticsearch /
Weaviate / Qdrant all use as default.

Prompt 16 — Batched classification
PROMPT TO PASTE INTO YOUR LLM
Add the main classify function:
async def classify_chunks(
texts: list[str],
batch_size: int = 10
) -> list[str]
Classifies a list of chunk texts. Returns list of category strings (one per text, in same
order). Each must be one of the four valid categories.
Implementation:
1. Process in batches of batch_size (default 10) to keep prompts under context window
2. For each batch, build a user message:
'Classify these chunks:\n\n[1] {first_100_chars_of_text_1}\n\n[2] {text_2}\n...'
3. Call: response = await _CLIENT.chat.completions.create(
model=_MODEL,
messages=[{'role': 'system', 'content': SYSTEM_PROMPT}, {'role': 'user', 'content':
user_msg}],
temperature=0, # deterministic
max_tokens=200, # categories are short
response_format={'type': 'json_object'} # if model supports it
)
4. Parse the response (JSON array). Validate each item is in _CATEGORIES; default to
'general' for invalid responses.
5. Append batch results to the overall list.
6. Add 0.5 second sleep between batches if rate-limited (Groq free tier is ~30 req/min).
Edge cases:
(cid:127) Empty texts list fi return []
(cid:127) Groq API error (timeout, rate limit) fi log error, return ['general'] * len(batch)
(cid:127) Response is not valid JSON fi log, return ['general'] * len(batch)
Add a test:

4. Verify accuracy is >70% (demo data is on-topic enough that an LLM should nail it)
Heads-up
