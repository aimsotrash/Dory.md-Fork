Prompt 8 — File parsing service
PROMPT TO PASTE INTO YOUR LLM
Create backend/integrations/file_parser.py with these functions:
 parse_markdown(file_bytes: bytes) -> str
Decodes UTF-8 and returns the text as-is. Strip leading/trailing whitespace.
 parse_pdf(file_bytes: bytes) -> str
Uses pypdf.PdfReader to extract text from all pages. Concatenate page texts with
'\n\n' between pages. Handle the case where text extraction returns None for a
page (treat as empty string).
 split_into_chunks(text: str, target_size: int = 350, overlap: int = 50) ->
list[str]
Splits text into chunks of approximately target_size characters. Splits ON
paragraph boundaries (double newline) when possible — never split mid-sentence
if avoidable. If a paragraph is longer than 2*target_size, split it by sentences
(split on '. '). Adds 'overlap' characters from the end of each chunk to the
start of the next to preserve context. Strips chunks of leading/trailing
whitespace. Discards chunks shorter than 30 characters as noise.
 detect_file_type(filename: str) -> str
Returns 'markdown' if filename ends with .md or .markdown, 'pdf' if .pdf, raises
ValueError otherwise.
Add a unit test in backend/tests/test_file_parser.py that:
1. Tests parse_markdown with a hardcoded markdown string
2. Tests split_into_chunks produces sensible chunks (right number, no empty
ones)
3. Tests detect_file_type for .md, .pdf, and an invalid extension
(Skip the PDF test for now — pypdf needs actual PDF bytes which is annoying to
mock.)
Prompt 9 — The /api/ingest endpoint
PROMPT TO PASTE INTO YOUR LLM
Create backend/routes/ingest.py with a router that handles POST /api/ingest.
Endpoint signature:
@router.post('/api/ingest')
async def ingest(files: list[UploadFile], db: AsyncSession = Depends(get_db))
Behavior:
1. For each file in files:
a. Read file bytes (await file.read())
b. Detect type via detect_file_type(file.filename)
c. Parse: parse_markdown or parse_pdf
d. Split into chunks via split_into_chunks
e. Create a Document row (source_type='file', source_name=file.filename)
f. Create Chunk rows in batch (call create_chunks_batch)
2. After all files processed, return:
{
'status': 'success',
'documents_imported': <count of docs>,
'chunks_count': <total chunks across all docs>,
'document_ids': [<list of doc IDs>]
}
3. If a file type is unsupported, skip it (don't fail the whole request) but
include the filename in a 'skipped' list in the response.
4. If parsing fails for a file, skip it similarly with 'errors' list in
response.
Wire up the router in backend/main.py: app.include_router(ingest_router).
Test it with curl:
curl -X POST http://localhost:8000/api/ingest -F 'files=@somefile.md'
Show me the curl test output after implementation.

PROMPT TO PASTE INTO YOUR LLM
Update the /api/ingest endpoint to use FastAPI BackgroundTasks for the

2. Compute retentions for all
3. Filter to chunks where retention < 0.5 and last_accessed > 24 hours ago
4. If empty list, return {'has_discovery': false}
5. Pick the chunk with lowest retention
6. Return:
{
'has_discovery': true,
'chunk': <ChunkOut as dict>,
'reason': 'You haven&#39;t looked at this in {N} days — likely 50% forgotten'
}
Where N = days since last_accessed.
Prompt 17 — Wire up /api/quiz
PROMPT TO PASTE INTO YOUR LLM
Update backend/routes/quiz.py:
POST /api/quiz/start:
1. Optional body param: category (e.g., 'technical'). If absent, use all chunks.
2. Get chunks (filtered by category if provided)
3. Compute retentions
4. Pick the 5 chunks with the lowest retention (these are the ones the user is
most likely to have forgotten — perfect for a quiz)
5. Call: from intelligence.quiz import generate_quiz_questions; questions =
await generate_quiz_questions(chunks)
6. Generate a quiz_id (uuid.uuid4().hex)
7. Cache the quiz in memory: store {quiz_id: {'questions': questions, 'chunks':
chunk_ids, 'created_at': now}}. Use a module-level dict — don't bother with
Redis for the hackathon.
8. Return: {'quiz_id': quiz_id, 'questions': [{'question', 'options'} for q in
questions]}
(Don't return correct_index — that's checked on submit)
POST /api/quiz/submit:
1. Body: {quiz_id: str, answers: list[int]}
2. Look up the quiz in memory cache
3. Compare answers to the stored correct_indices
4. For each correct answer, also update the corresponding chunk's access (call
update_chunk_access — answering correctly counts as a review)
5. Return: {'score': N/5, 'correct_count': N, 'detail': [{question_id,
was_correct, correct_index}]}
Prompt 18 — Wire up the post-ingest pipeline
PROMPT TO PASTE INTO YOUR LLM
Update backend/services/post_ingest.py to call Person 2's modules.
async def post_ingest_pipeline(chunk_ids: list[str]):
1. Open a new db session (background tasks need their own)
2. Fetch the chunks: chunks = await get_chunks_by_ids(db, chunk_ids)
3. Get embeddings: embeddings = embed_chunks([c.content for c in chunks])
4. Add to ChromaDB: from intelligence.embedder import store_embeddings;
store_embeddings(chunks, embeddings)
5. Classify: categories = await classify_chunks([c.content for c in chunks])
6. Update each chunk's category in db: for chunk_id, cat in zip(chunk_ids,
categories): await update_chunk_category(db, chunk_id, cat)
7. Log completion: print(f'Embedded and classified {len(chunks)} chunks')
Same fallback rule: if any P2 module fails, log error and don't fail the whole
pipeline. The chunk is in SQLite already — partial state is fine.
